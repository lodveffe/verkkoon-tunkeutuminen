## b) Tutustu APDU komentojen rakenteeseen (voit käyttää tekoälyä tutustumiseen)

APDU toimii älykorteissa samalla tavoin kuin HTTP-pyyntö ja vastaus verkkomaailmassa: lukija lähettää käskyn (Command APDU) ja kortti vastaa datalla sekä tilakoodilla (Response APDU). (Passninja, 2025)
Tähän väitteeseen ChatGPT loi tämmöisen vertauksen:

HTTP GET = “Hae tämä resurssi”

APDU SELECT = “Valitse tämä tiedosto/sovellus kortista”

HTTP status 200 OK = onnistui

APDU status 90 00 = onnistui

HTTP 404 Not Found = ei löydy

APDU 6A 82 = file not found

Kyselin lisää tekoälyltä APDUsta, kun aika huonoja artikkeleita tulee vastaan mun google-hauilla. Eli APDU-komento koostuu command APDUsta ja response APDUsta:

### Command APDU (lukijan lähettämä komento kortille)

    CLA | INS | P1 | P2 | [Data]    

- __CLA__ = komentoluokka
- __INS__ = itse komento
- __P1/P2__ = parametreja
- __Data__ = lisätietoa, esim. mitä tiedostoa haetaan

### Response APDU (kortin vastaus komentoon)

    [Data] | SW1 SW2

- __Data__ = palautettu tieto (jos on)
- __SW1 SW2__ = kahden tavun statuskoodi

Ei pidä unohtaa että APDU ei kuitenkaan ole protokolla, vaan viestirakenne ISO 7816‑4 ‑standardissa. (Wikipedia, 2025)
